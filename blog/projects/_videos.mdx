   2. Describe the GH action
      1. Create a release when you merge a commit into `main`.
         1. Creates a tag with the release number   => `x.y.z` (this follows semantic versioning)
         2. Creates a record in a changelog file    => `CHANGELOG.md`
   3. Out of the box features
2. Explain the requirements
   1. This needs the last commit to be squashed
   2. It only expected one commit message
   3. [conventional commits](https://www.conventionalcommits.org/en/v1.0.0/)
   4. [semantic versioning](https://semver.org)
